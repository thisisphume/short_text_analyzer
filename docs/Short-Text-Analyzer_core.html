Creating Short-text Analyzer

There are 3 mains process in the short-text analyzer which are:

  1. Validating the input: to ensure the data has the right format.
  2. Preprocessing the input:
    • translate the non-english text to english
    • embeddding the text using pre-trained BERT model
  3. Analyzing the input and get the result:
    • Sentiment Score
    • Subjectivity Score
    • Clustering via HDBSCAN or KMeans
  4. Plotting the result: plotting the 2D representation of the input coloured by assigned cluster.

embeddingComments: mbedding the text via BERT and using UMAP for dimension reduction:

to transforms text into a numerical representationto (text embedding).

There are two steps:1. Embedding: using pre-trained BERT model 2. Dimension Reduction: using UMAP to reduce the number of dimensions.

There are 3 different outputs:

  1. embeddingRaw: this is the raw embedding (without going through the dimension reduction).
  2. embeddingForVisualize: the UMAP was used to reduce to two dimensions so it can be used to plot on a 2D-plane.
  3. embeddingForCluster: the UMAP was used to reduce to five dimension. This is prefered for clustering purpose because the data doesn't get compress as much.

Reference:

  • https://umap-learn.readthedocs.io/en/latest/clustering.html#traditional-clustering

Clustering via HDBSCAN and KMeans

Here we consider two different clustering algorithms, HDBSCAN and KMeans.

  1. HDBSCAN is one of the most advanced density based techniques. Number of cluster are selected based on the min_cluster_size paramnumber that has the lowest number of outlier.
  2. KMeans is one of the most popular clustering algorithm. Unlike HDBSCAN, the number of clusters must be specified by the user.

Both methods were used so there is some flexibility on the number of clusters that the user wants. The visualization can be used to help determine if we have the appropriate number of clusters.
